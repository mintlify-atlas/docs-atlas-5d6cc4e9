## Language Features

Each supported language includes:

- **Voice Cloning**: Train custom voice models from audio samples
- **Text-to-Speech**: Generate speech from text input
- **Natural Prosody**: Contextually appropriate intonation and rhythm
- **Accent Preservation**: Maintains speaker's accent characteristics
- **Multi-lingual Models**: Single model can handle all supported languages

## Voice Training with Language Selection

### Training Workflow

When training a voice model, the language parameter determines pronunciation rules and phonetic processing:

1. **Prepare Audio Sample**
   - Record or provide audio in target language
   - Minimum 10 seconds, recommended 30-60 seconds
   - Clear audio with minimal background noise

2. **Extract Audio from Video**
   
   Separate your training video:
   ```bash
   # This step is handled automatically by the client
   # Produces: silent_video.mp4 + audio.wav
   ```

3. **Place Audio in Training Directory**
   
   **Windows:**
   ```
   D:\duix_avatar_data\voice\data\your_audio.wav
   ```
   
   **Linux:**
   ```
   ~/duix_avatar_data/voice/data/your_audio.wav
   ```

4. **Train Voice Model with Language Parameter**
   
   The training API includes language specification:
   
   ```javascript
   // Model training request
   const response = await fetch('http://127.0.0.1:18180/v1/train', {
     method: 'POST',
     body: formData
   });
   
   // Response includes reference audio and text
   {
     "uuid": "unique-voice-id",
     "asr_format_audio_url": "/path/to/processed/audio.wav",
     "reference_audio_text": "Transcribed text in target language"
   }
   ```

## Text-to-Speech Synthesis

### API Usage by Language

The TTS API endpoint accepts text in any supported language:

```javascript
POST http://127.0.0.1:18180/v1/invoke

Content-Type: application/json

{
  "speaker": "unique-voice-id",
  "text": "Text in any supported language",
  "format": "wav",
  "topP": 0.7,
  "max_new_tokens": 1024,
  "chunk_length": 100,
  "repetition_penalty": 1.2,
  "temperature": 0.7,
  "reference_audio": "/path/to/voice/sample.wav",
  "reference_text": "Reference text in same language"
}
```

## Language Detection

Duix Avatar's TTS engine automatically detects the language of input text:

- **Auto-detection**: Text language is identified from Unicode characters
- **Mixed Languages**: Can handle code-switching within reasonable limits
- **Fallback**: Defaults to reference audio language if uncertain

<Warning>
**Best Practice**: For optimal results, keep text in a single language. Mixed-language content may have inconsistent pronunciation.
</Warning>

## Character Encoding

Ensure proper character encoding for non-Latin scripts:

### API Requests

```javascript
// JavaScript/Node.js
const response = await fetch('http://127.0.0.1:18180/v1/invoke', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json; charset=utf-8'
  },
  body: JSON.stringify({
    text: "你好世界",  // UTF-8 encoded
    // ... other parameters
  })
});
```

### File Encoding

When preparing text files:

```bash
# Verify UTF-8 encoding
file -i your_script.txt

# Convert to UTF-8 if needed
iconv -f ISO-8859-1 -t UTF-8 input.txt > output.txt
```

## Quality Optimization

### Training Data Quality

For best results across all languages:

1. **Audio Duration**: 30-60 seconds minimum
2. **Sample Rate**: 44.1kHz or higher
3. **Format**: WAV, uncompressed
4. **Content**: Native speaker with clear articulation
5. **Background**: Minimal noise, no music
6. **Speech Style**: Natural conversational tone

### Text Preparation

<Accordion title="English">
  - Use proper capitalization and punctuation
  - Spell out numbers and abbreviations
  - Include natural pauses with commas and periods
</Accordion>

<Accordion title="Chinese">
  - Use simplified or traditional characters consistently
  - Include proper punctuation (。，！？)
  - Avoid mixing with excessive English words
</Accordion>

<Accordion title="Japanese">
  - Use natural kanji/kana mix
  - Include proper particles (は、が、を、に)
  - Use Japanese punctuation (。、)
</Accordion>

<Accordion title="Arabic">
  - Include diacritical marks for better pronunciation
  - Use proper Arabic punctuation
  - Ensure RTL text direction in editors
</Accordion>

## Multilingual Scenarios

### Creating Multiple Language Avatars

Create separate voice models for each language:

```javascript
// Train English voice
const enVoice = await trainVoice({
  audio: 'english_sample.wav',
  language: 'en'
});

// Train Chinese voice
const zhVoice = await trainVoice({
  audio: 'chinese_sample.wav',
  language: 'zh'
});

// Use appropriate voice for each video
const englishVideo = await synthesizeVideo({
  speaker: enVoice.uuid,
  text: 'Hello world',
  video_url: 'avatar.mp4'
});

const chineseVideo = await synthesizeVideo({
  speaker: zhVoice.uuid,
  text: '你好世界',
  video_url: 'avatar.mp4'
});
```

### Same Avatar, Multiple Languages

Use the same video model with different voice models:

1. Train separate voice models for each language
2. Use the same source video for all
3. Generate language-specific audio
4. Synthesize videos with matching language audio

## Troubleshooting

<Accordion title="Incorrect pronunciation">
  **Symptoms:**
  - Words pronounced incorrectly
  - Wrong intonation or accent
  
  **Solutions:**
  1. Verify training audio is in the same language as synthesis text
  2. Ensure reference text matches reference audio language
  3. Use higher quality training samples (60+ seconds)
  4. Check text encoding (UTF-8 required)
</Accordion>

<Accordion title="Garbled characters in output">
  **Symptoms:**
  - Non-Latin characters appear as �� or boxes
  - Text not displaying correctly
  
  **Solutions:**
  1. Verify UTF-8 encoding:
  ```bash
  file -i your_file.txt
  ```
  
  2. Convert to UTF-8:
  ```bash
  iconv -f YOUR_ENCODING -t UTF-8 input.txt > output.txt
  ```
  
  3. Set correct Content-Type header:
  ```javascript
  headers: {'Content-Type': 'application/json; charset=utf-8'}
  ```
</Accordion>

<Accordion title="Mixed language synthesis issues">
  **Symptoms:**
  - Unnatural pronunciation when mixing languages
  - Voice switches between different styles
  
  **Solutions:**
  1. Avoid mixing languages in single text block
  2. Train separate models for each language
  3. Split content into language-specific segments
  4. Use language-appropriate punctuation
</Accordion>
