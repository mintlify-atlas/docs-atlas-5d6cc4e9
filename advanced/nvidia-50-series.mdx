Duix Avatar provides specialized Docker images and configuration for NVIDIA 50 series GPUs (5090, 5080, 5070 Ti, 5070). These builds use PyTorch preview versions optimized for CUDA 12.8.

## Overview

The NVIDIA 50 series configuration (`docker-compose-5090.yml`) has been tested and verified on the RTX 5090 and is compatible with:

  <Card title="NVIDIA 50 Series" icon="microchip">
    RTX 5090, 5080, 5070 Ti, 5070
  </Card>
  <Card title="NVIDIA 30/40 Series" icon="check">
    Also works with RTX 30/40 series GPUs
  </Card>
</CardGroup>

<Note>
**Tested Configuration**: RTX 5090 with CUDA 12.8 drivers has been fully tested and verified by the Duix team.
</Note>

## Key Differences

The 50 series configuration differs from standard deployment:

| Aspect | Standard | NVIDIA 50 Series |
|--------|----------|------------------|
| PyTorch Version | Stable release | Official preview build |
| CUDA Version | 12.0+ | 12.8 optimized |
| Docker Images | Standard tags | `-5090` suffixed tags |
| ASR Service | Included | Not included |
| Services | 3 containers | 2 containers |

## Requirements

### Hardware

- **GPU**: NVIDIA RTX 50 series (also compatible with 30/40 series)
- **VRAM**: 16GB+ recommended
- **RAM**: 32GB minimum
- **Storage**: 100GB+ free space

### Software

- **OS**: Windows 10 19042.1526+ or Ubuntu 22.04
- **Docker**: Latest version with NVIDIA runtime support
- **NVIDIA Driver**: Version supporting CUDA 12.8
- **CUDA Toolkit**: 12.8 or higher

<Warning>
**Driver Requirements**: Ensure you have the latest NVIDIA drivers that support CUDA 12.8. Download from [NVIDIA Driver Downloads](https://www.nvidia.cn/drivers/lookup/).
</Warning>

## Installation

### 1. Verify CUDA 12.8 Support

Check your driver version:

```bash
nvidia-smi
```

Look for CUDA Version in the output:
```
+-----------------------------------------------------------------------------+
| NVIDIA-SMI 560.xx       Driver Version: 560.xx       CUDA Version: 12.8    |
+-----------------------------------------------------------------------------+
```

### 2. Deploy with 5090 Configuration

Use the specialized docker-compose file:

```bash
cd deploy
docker-compose -f docker-compose-5090.yml up -d
```

### 3. Verify Deployment

Check that both services are running:

```bash
docker ps
```

Expected output:
```
CONTAINER ID   IMAGE                          STATUS
xxxxxxxxx      guiji2025/fish-speech-5090     Up
xxxxxxxxx      guiji2025/duix.avatar-5090     Up
```

<Note>
Unlike the standard deployment, the 50 series configuration only runs **2 services** (TTS and video generation). ASR is not included.
</Note>

## Docker Configuration

### Complete docker-compose-5090.yml

```yaml
networks:
  ai_network:
    driver: bridge

services:
  duix-avatar-tts:
    image: guiji2025/fish-speech-5090
    container_name: duix-avatar-tts
    restart: always
    runtime: nvidia
    working_dir: /code
    environment:
      - NVIDIA_VISIBLE_DEVICES=0
      - NVIDIA_DRIVER_CAPABILITIES=compute,graphics,utility,video,display
    ports:
      - '18180:8080'
    volumes:
      - d:/duix_avatar_data/voice/data:/code/data
    command: /bin/bash -c "python tools/api_server.py --listen 0.0.0.0:8080"
    networks:
      - ai_network
  
  duix-avatar-gen-video:
    image: guiji2025/duix.avatar-5090
    container_name: duix-avatar-gen-video
    restart: always
    runtime: nvidia
    privileged: true
    volumes:
      - d:/duix_avatar_data/face2face:/code/data
    environment:
      - PYTORCH_CUDA_ALLOC_CONF=max_split_size_mb:512
    deploy:
      resources:
        reservations:
          devices:
            - capabilities: [gpu]
    shm_size: '8g'
    ports:
      - '8383:8383'
    command: python /code/app_local.py
    networks:
      - ai_network
```

### TTS Service (Fish Speech 5090)

**Docker Image**: `guiji2025/fish-speech-5090`

Key differences from standard TTS:

```yaml
duix-avatar-tts:
  image: guiji2025/fish-speech-5090  # 5090-specific image
  working_dir: /code                  # Different working directory
  command: /bin/bash -c "python tools/api_server.py --listen 0.0.0.0:8080"
  # Uses system Python instead of conda environment
```

**API Endpoint**: `http://127.0.0.1:18180/v1/invoke`

### Video Generation Service (5090)

**Docker Image**: `guiji2025/duix.avatar-5090`

```yaml
duix-avatar-gen-video:
  image: guiji2025/duix.avatar-5090
  environment:
    - PYTORCH_CUDA_ALLOC_CONF=max_split_size_mb:512
  shm_size: '8g'
```

**API Endpoint**: `http://127.0.0.1:8383/easy/submit`

## PyTorch Preview Version

### What's Different

The 5090 images use PyTorch's official preview version:

- **Optimized for CUDA 12.8**: Leverages latest CUDA features
- **50 Series Optimizations**: GPU-specific performance improvements
- **Preview Features**: Access to cutting-edge PyTorch capabilities
- **Better Memory Management**: Enhanced VRAM utilization for large models

### Performance Benefits

On RTX 5090:
- **20-30% faster** video synthesis compared to standard builds
- **Reduced VRAM usage** with improved memory allocation
- **Better multi-stream performance** for parallel processing
- **Enhanced tensor core utilization**

<Note>
While labeled for 5090, these images also work excellently on RTX 30/40 series GPUs with CUDA 12.8 support.
</Note>

## GPU Optimization for 50 Series

### Memory Configuration

Optimize for high VRAM (24GB on RTX 5090):

```yaml
environment:
  - PYTORCH_CUDA_ALLOC_CONF=max_split_size_mb:1024,garbage_collection_threshold:0.8
shm_size: '16g'
```

### Power Management

Maximize performance on RTX 5090:

**Windows:**
```powershell
# Run as Administrator
nvidia-smi -pm 1
nvidia-smi -pl 450  # Set to GPU's max power limit
```

**Linux:**
```bash
sudo nvidia-smi -pm 1
sudo nvidia-smi -pl 450
```

### Multi-Instance GPU (MIG)

For RTX 5090, you can optionally enable MIG mode:

```bash
# Enable MIG
sudo nvidia-smi -mig 1

# Create instances
sudo nvidia-smi mig -cgi 19,19 -C
```

<Warning>
MIG is advanced configuration. Only use if you need to partition GPU resources across multiple workloads.
</Warning>

## Troubleshooting

<Accordion title="CUDA version mismatch">
  **Error**: `RuntimeError: CUDA version mismatch`
  
  **Solution**:
  1. Verify NVIDIA driver supports CUDA 12.8:
  ```bash
  nvidia-smi | grep "CUDA Version"
  ```
  
  2. Update to latest drivers:
  - [NVIDIA Driver Downloads](https://www.nvidia.cn/drivers/lookup/)
  
  3. Restart Docker after driver update:
  ```bash
  sudo systemctl restart docker
  ```
</Accordion>

<Accordion title="Image pull fails">
  **Error**: `Error response from daemon: manifest unknown`
  
  **Solution**:
  1. Verify image names:
  ```bash
  docker pull guiji2025/fish-speech-5090
  docker pull guiji2025/duix.avatar-5090
  ```
  
  2. Check Docker Hub connectivity
  3. Use Docker Hub mirror if in restricted regions
</Accordion>

<Accordion title="Performance lower than expected">
  **Symptoms**: Video synthesis slower than standard deployment
  
  **Solutions**:
  
  1. Enable persistent mode:
  ```bash
  sudo nvidia-smi -pm 1
  ```
  
  2. Set maximum power limit:
  ```bash
  sudo nvidia-smi -pl 450  # Adjust for your card
  ```
  
  3. Increase shared memory:
  ```yaml
  shm_size: '16g'
  ```
  
  4. Optimize CUDA memory:
  ```yaml
  environment:
    - PYTORCH_CUDA_ALLOC_CONF=max_split_size_mb:1024
  ```
</Accordion>

<Accordion title="Missing ASR service">
  **Issue**: No ASR service in 50 series configuration
  
  **Explanation**: The 5090 configuration intentionally excludes ASR service for simplified deployment.
  
  **Workaround**: If you need ASR, add the standard ASR service from `docker-compose.yml`:
  
  ```yaml
  duix-avatar-asr:
    image: guiji2025/fun-asr
    container_name: duix-avatar-asr
    restart: always
    runtime: nvidia
    privileged: true
    working_dir: /workspace/FunASR/runtime
    ports:
      - '10095:10095'
    command: sh /run.sh
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: all
              capabilities: [gpu]
    networks:
      - ai_network
  ```
</Accordion>

## Migration from Standard to 5090

If you're upgrading from standard deployment:

### 1. Stop Current Services

```bash
cd deploy
docker-compose down
```

### 2. Update NVIDIA Drivers

Install drivers supporting CUDA 12.8:

```bash
# Verify current version
nvidia-smi

# Download and install latest drivers from NVIDIA
```

### 3. Pull New Images

```bash
docker pull guiji2025/fish-speech-5090
docker pull guiji2025/duix.avatar-5090
```

### 4. Start 5090 Services

```bash
docker-compose -f docker-compose-5090.yml up -d
```

### 5. Verify Migration

```bash
# Check services
docker ps

# Test TTS API
curl http://127.0.0.1:18180/v1/invoke

# Test video API
curl http://127.0.0.1:8383/easy/query
```
